bind(rootNodes, selector)
Walk DOM tree(s) and extract model data, allowing for subsequent updating.

After execution, each value element encountered will have a corresponding representation in the model. Altering such model values and then calling updateDOM(...) will refresh their displayed value.

The DOM tree(s) to walk can be specified in exactly one of three ways:

  1. Horn will automatically find all DOM trees that have a root indicator.
  2. Passing a collection of DOM elements as args.rootNodes.
  3. Passing a JQuery selector string as args.selector.

Before a value is stored in the model it is converted to its String representation. Alternatively, applications can register converter functions to override this default behaviour.

If later, model to DOM updates are not required, the alternative yet otherwise identical function, Horn#load should be used.

Parameters:
{Object} rootNodes Optional
a collection of DOM nodes to bind from, overrides the default node selection mechanism
{String} selector Optional
a jQuery DOM node selector for nodes to bind from,
Returns:
the updated model
See:

traverse(value, callback, path, context, propName)
Traverses an object graph and executes a callback function for each value encountered.

Object and Array types are considered containers of values rather than values themselves.

For any model value, the corresponding equivalent Horn property path is constructed and reported to the callback function. An optional property path stem may be supplied which will be prepended to any such path.

Parameters:
value
the root of the object graph to traverse
{Function} callback
a callback function that will be executed for each value of the object graph encountered, it should have the following signature: ( path, value, context, propName) where, 'path' is the value's Horn path within its container, 'value' is the current value found, 'context' is the object in which the value can be found and 'key' is the property name within 'context' that the value can be found.
{String} path Optional
an optional property path that will be prefixed to every callback reported property path constructed
context Optional
internal use only - no value required
propName Optional
