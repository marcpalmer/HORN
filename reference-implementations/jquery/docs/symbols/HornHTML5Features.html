Class Detail

HornHTML5Features()
A Horn delegate implementation that extracts Horn data from custom HTML5 data attributes.

Please refer to our online documentation for fuller details http://horn.io/.

Use this function to create new HornHTML5Features instances, thus: var hornHTML5Features = new HornHTML5Features();.

Set it to use on a horn instance, horn.delegate( hornHTML5Features);.

Returns:
{HornHTML5Features} a newly initialised HornHTML5Features instance

    Determine if a node declares the JSON indicator.

    Nodes that declare this indicator are implicitly value nodes and contain literal JSON encoded as the single text element body value child of declaring elements.

    Parameters:
    {Element} node
    the element that may declare the Horn JSON indicator
    Returns:
    true if 'node' declares the Horn JSON indicator
  • hasRootIndicator(node)
    Determine if a given node possesses a Horn root node indicator (the data attribute 'horn').

    In this implementation we return true if 'node' declares a data-attribute named 'horn' with any value whatsoever.

    Parameters:
    {Element} node
    the node to examine as to declaring a root indicator
    Returns:
    true if 'node' does possess a Horn root node indicator (of any value), false otherwise.
  • {String|Boolean} pathIndicator(node)
    Extracts and returns the Horn path indicator for a given node.

    In this, the HTML5 implementation, nodes can declare paths using data-attributes named, either 'horn', OR 'json'.

    Paths are given in full JavaScript-like syntax so for example, a.b[a].x[2][3

    Parameters:
    {Element} node
    the node from which to extract the path indicator
    Returns:
    {String|Boolean} if 'node' does have a Horn path indicator, it is returned in String form, else Boolean false is be returned
  • rootNodes(args)
