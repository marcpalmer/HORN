  • bind(args)
    Walk DOM tree(s) and extract model data, allowing for subsequent model to DOM updates.

    After execution, each Horn value element encountered will have a corresponding representation in the model. Altering such model values and then calling updateDOM(...) will refresh their display values.

    The DOM tree(s) to walk may be specified in exactly one of two ways:

    1. Horn will automatically find all DOM trees that have a root indicator.
    2. Passing in 'args.nodes', a collection of DOM elements or a jQuery node selector.

    Before a value is stored in the model it is converted to its String representation. Alternatively, applications can register converter functions to override this default behaviour.

    If model to DOM updates are not required, the alternative yet otherwise identical function, Horn#load should be used (or the 'readOnly' option).

    Parameters:
    {Object} args
    {Element|String} args.nodes Optional
    DOM nodes or a jQuery selector
    {String} args.pathStem Optional
    a property path prefix that will be appended the paths of value nodes' property paths before storage in the model
    Returns:
    the updated model
    See:
    Horn#option for the detailing of converter functions
    Horn#load
  • bindTo(args)
    Walks a tree of cloned (or selected) DOM nodes, binding them to model elements and populating their display values.

    During the walk, the property path for each DOM value node visited is inspected and if there is a corresponding model element with the same path it will be bound to the DOM node and the DOM node will receive its value.

    For each DOM value node encountered, the optional 'args.pathStem' will be added to its property path before binding to a model value. Allowing nodes derived from a single template to be bound to multiple model locations.

    Model to DOM data conversions are performed using this function with the type parameter value 'toText'.

    Parameters:
    {Object} args
    {Element|String} args.node Optional
    a jQuery node or selector - nodes selected by this argument will not be cloned.
    {Element|String} args.template Optional
    jQuery node or selector - these nodes will be cloned and the root node will have its 'id' attribute removed and possibly set to 'args.newID'
    {String} args.id Optional
    the new 'id' attribute value applied to nodes cloned or selected using 'args.template'
    {String} args.pathStem Optional
    a property path prefix appended to all walked DOM nodes before binding to model values
    Returns:
    the newly cloned and populated template

  • option(optionName, value)
    Get an option's value by name, or set an option's value by name.

    If no value is provided, the value of the named option is returned, otherwise the return value is undefined.

    The following options are currently supported:

    • defaultModel - For setting an explicit default model (Object or Array)

    • readOnly - If set to true, the automatic extraction at startup will call load() instead of bind() so there is no two-way binding to the DOM.

    • converter - A function with the following signature, (args) that will be called upon to convert values under the following scenarios:

      • DOM to model :- when values are extracted from the DOM for storage in the model (functions Horn#bind, Horn#load) the convert function will be called with 'type' parameter value 'fromText'.

      • Model to DOM :- when values read from the model are destined for the DOM, (functions Horn#updateDOM, Horn#bindTo) the convert function will be called with 'type' parameter value 'toText'.

      • JSON to model :- when JSON values are extracted from the DOM for model storage (functions Horn#bind, Horn#load), the convert function will be called with 'type' parameter value 'fromJSON'.

      The complete set of arguments passed to the convert function are namely:

      • args.type :- the conversion 'type' as described above.

      • args.value :- the value for conversion. The JavaScript type is dependent on the conversion being performed (String for 'fromText' conversions and the corresponding model value's type in all others).

      • args.path :- the property path for the DOM or model value.

      • args.node :- the DOM node that generated the value.

      Converter functions can return undefined in which case the value (all such values) will be treated as String types - the same behaviour as having no converter function registered.

      Only a single converter function can be registered, with any invocation overwriting any previously registered function.

    Parameters:
    {Object} optionName
    the name of the option to set
    {Object} value Optional
    the value to set
    Returns:
    the value of the named option if no value supplied else undefined

  • traverse(value, callback, path, context, propName)
    Traverses an object graph and executes a callback function for each value encountered.

    Object and Array types are considered containers of values rather than values themselves.

    For any model value, the corresponding equivalent Horn property path is constructed and reported to the callback function. An optional property path stem may be supplied which will be prepended to any such path.

    Parameters:
    value
    the root of the object graph to traverse
    {Function} callback
    a callback function that will be executed for each value of the object graph encountered, it should have the following signature, ( path, value, context, propName), where 'path' is the value's property path within its container, 'value' is the current value found, 'context' is the object in which the value can be found and 'key' is the property name within 'context' that the value can be found.
    {String} path Optional
    an optional property path that will be prefixed to every callback reported property path constructed
    context Optional
    internal (recursive) use only - do not supply a value
    propName Optional
    internal (recursive) use only - do not supply a value
